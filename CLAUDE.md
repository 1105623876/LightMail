# CLAUDE.md

## 项目背景

本目录对应 2026 年计算机网络实践的软件设计实验。

本项目已确定选题为邮件系统。

邮件系统要求：

- 包括 SMTP 客户端和 POP3 客户端
- 支持邮件编写、发送、接收、阅读、删除等基本功能
- Windows 客户端
- 必须使用 socket 方式编程，从创建 socket、建立 TCP 连接开始实现 SMTP、POP3 协议功能

技术栈已确定为 Python。优先使用标准库实现：`socket` / `ssl` 实现 SMTP、POP3 协议通信；GUI 使用 Tkinter ttk；本地账号配置和邮件缓存使用 SQLite。优先适配 QQ 邮箱，使用授权码而不是登录密码。

## 当前待确认

- 小组成员与分工尚未确定：实验报告中必须注明每个人完成的具体任务，并注明源代码由哪位成员完成。

## 开发约束

- 重点满足课程实验要求，不做过度工程化。
- 只实现选题要求的最小可用功能，除非用户明确要求扩展。
- 网络协议功能应基于 socket 直接实现，不能只调用现成高级 FTP/SMTP/POP3 客户端库来替代协议实现。
- Python 中不要用 `smtplib`、`poplib` 等高级邮件库替代协议实现；可使用 `email` 标准库处理 MIME 编解码。
- 客户端目标环境为 Windows。
- 如果需要新增依赖或全局安装工具，先向用户确认。
- 代码风格保持简洁清晰，避免无用注释和无关重构。
